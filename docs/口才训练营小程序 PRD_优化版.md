# 口才训练营小程序 PRD（优化版）

---

## 1. 文档概述

### 1.1 产品基础信息

| 维度 | 内容 |
|------|------|
| **产品名称** | 口才训练营小程序 |
| **产品Slogan** | 每日科学练口才，自信表达看得见 |
| **产品定位** | 轻量化微信端口才专项训练小程序。面向零基础及有表达提升需求的用户，提供**分阶口才练习、全场景话术实战、每日打卡监督、AI文案辅助、AI语音评测反馈**五大核心能力 |
| **运行环境** | 微信小程序 |
| **目标终端** | 全尺寸手机端，适配 iOS / Android |
| **一期商业模式** | 完全免费，无付费模块（二期预留会员/付费课程接口） |

### 1.2 核心痛点与解决方案

| 用户痛点 | 产品解决方案 |
|----------|-------------|
| 当众发言紧张、怯场 | 演讲脱敏训练 + 即兴表达练习，阶梯式暴露练习 |
| 说话无逻辑、无条理 | 结构化表达模板（PREP/金字塔/时间线）+ AI逻辑点评 |
| 缺少话术素材和演讲稿 | AI文案生成（Qwen）+ 内置话术模板库 |
| 无法坚持练习、三分钟热度 | 每日打卡体系 + 成长目标 + 微信服务通知提醒 |

### 1.3 技术决策总结

| 决策项 | 选型 | 说明 |
|--------|------|------|
| 前端框架 | **uni-app (Vue3)** | 一套代码编译微信小程序，生态成熟 |
| 后端框架 | **Python Flask** | 轻量灵活，和AI生态衔接好 |
| 数据库 | **MySQL** (阿里云RDS) | 结构化数据，关系型查询需求多 |
| 云服务 | **阿里云全家桶** | OSS存录音、ECS部署后端、RDS数据库 |
| AI文案模型 | **Qwen（通义千问）** | 阿里云百炼平台调用 |
| 语音评测 | **阿里云语音评测API** | 普通话发音评分（准确度/流利度/完整度） |
| AI反馈生成 | **Qwen** | 接收评测数据，生成个性化改进建议 |
| 录音存储 | **阿里云OSS** | 云端保存，跨设备同步 |
| 音频范本 | **TTS合成为主** | 一期快速上线，后期迭代真人录制 |

## 2. 目标用户画像

### 2.1 四类核心用户

#### 👔 职场人群（主力用户）
- **场景**：工作汇报、面试答辩、会议发言、跨部门沟通
- **痛点**：上台紧张声音发抖、表达缺乏逻辑结构、被领导质疑"说不清楚"
- **使用动机**：升职加薪、跳槽面试、日常工作沟通
- **使用时段**：通勤路上、午休碎片时间

#### 🎓 学生人群
- **场景**：课堂发言、社团竞选、毕业答辩、考研复试
- **痛点**：怯于当众开口、语言组织能力弱、缺少实战练习机会
- **使用动机**：学业需求、社交自信
- **使用时段**：晚自习后、周末

#### 📱 自媒体/主播人群
- **场景**：短视频口播、直播带货、内容创作
- **痛点**：缺少话术脚本、直播互动生硬、口播表现力不足
- **使用动机**：涨粉变现、提升直播转化
- **使用时段**：开播前准备、内容创作期

#### 💬 普通大众
- **场景**：日常社交、家庭沟通、朋友聚会
- **痛点**：表达生硬不自然、不会找话题、缺少沟通自信
- **使用动机**：改善人际关系、提升个人魅力
- **使用时段**：随时碎片时间

### 2.2 典型用户故事

> **小王（职场新人，26岁）**：下个月要做转正述职汇报，一上台就脑子空白。在口才训练营练了2周"工作汇报"模板，每天跟读练习15分钟，汇报当天虽然还有点紧张，但结构化表达很顺畅，顺利转正。现在每天打卡练即兴表达，想进一步提升。

> **小美（大学生，20岁）**：学生会竞选演讲，准备了很久但不知道讲得好不好。用AI文案生成了竞选演讲稿，录音练习后收到了AI反馈："语速偏快，激情处可以适当停顿"，调整后演讲效果很好，成功当选。

## 3. 用户核心流程

### 3.1 首次用户体验流程

```
微信搜索/扫码进入 → 微信一键授权登录
    ↓
首页展示新手引导（3张引导卡片，介绍4大功能）
    ↓
引导进入"7天零基础口才入门训练营"
    ↓
完成第一个练习（朗读短句30秒）→ 收到AI评分 + 鼓励反馈
    ↓
自动加入每日打卡计划 → 首页展示明日任务预告
```

### 3.2 每日核心练习流程

```
收到微信服务通知（可选）→ 打开小程序
    ↓
首页顶部展示"今日待练3个任务"
    ↓
进入每日打卡页 → 逐一完成3个任务（每个≤5分钟）
    ├── 任务1：跟读朗读（播放范本→录音跟读→AI评分）
    ├── 任务2：短句表达（给定场景→组织语言→录音→AI评测）
    └── 任务3：即兴口述（随机话题→即兴发挥→AI反馈）
    ↓
3个任务全部完成 → 打卡成功 → 日历标绿
    ↓
赠送1次AI文案生成次数 + 更新成长数据
```

### 3.3 异常流程处理

| 异常场景 | 处理方案 |
|----------|----------|
| 录音中网络中断 | 本地缓存录音文件，网络恢复后自动上传，提示用户"录音已本地保存" |
| AI评测超时/失败 | 友好提示"评分服务繁忙，请稍后重试"，本次录音计为完成（降级策略） |
| AI文案生成失败 | 提示"生成失败，本次不消耗次数"，支持一键重试 |
| 未授权录音权限 | 弹窗引导用户开启，"开启录音才能进行口才练习哦~" |
| 小程序切后台 | 录音继续录制，回来后可继续完成 |
| 存储空间不足 | 提示清理缓存，旧录音可手动删除 |

#### 每日三大任务设计

| 任务 | 类型 | 时长 | 核心目标 | AI评测维度 |
|------|------|------|----------|-----------|
| **跟读朗读** | 被动练习 | ≤3min | 纠正发音、培养语感 | 发音准确度、声调、流利度 |
| **短句表达** | 半结构化 | ≤5min | 场景化表达、逻辑组织 | 内容完整性、逻辑性、表达流畅 |
| **即兴口述** | 主动发挥 | ≤5min | 快速反应、思维敏捷 | 内容质量、结构逻辑、表达技巧 |

#### 打卡判定规则

```
每日3个任务全部完成 → 当日打卡成功
  ├── 单个任务判定依据：录音时长 ≥ 最低时长要求 + AI评分 ≥ 基准线
  ├── 同步完成（≥24:00前）：正常打卡
  ├── 补打卡（次日）：允许补签前一天，但标记为"补签"（黄色而非绿色）
  └── 免打扰模式：用户可设置"今日休息"，不产生漏打卡记录

特殊情况降级：
  - AI评测服务不可用 → 仅判断录音时长，自动标为完成
  - 用户录音时长不足 → 提示"请至少练习1分钟才计入完成哦~"
```

#### 口才能力维度评分

```
评分维度（5维雷达图，满分100）：

    发音准确度
        /\
       /  \
  逻辑力   流利度
      \  /
       \/
   表现力 ── 内容力

├── 发音准确度：基于阿里云语音评测的声母/韵母/声调准确率
├── 流利度：基于语速均匀度、卡顿次数、词间停顿
├── 内容力：基于Qwen对内容完整性、深度的语义分析
├── 逻辑力：基于Qwen对表达结构的分析（PREP/因果/时间线）
└── 表现力：基于音量变化、重音运用、情感饱满度
```

| 规则 | 说明 |

#### AI文案生成技术方案

```
用户输入 → Flask后端 → 组装Prompt → 调用Qwen API → 返回文案

Prompt模板（以"演讲文案"为例）：
---
你是一位专业的演讲稿撰写专家。请根据以下信息生成一篇演讲稿：

【主题】{用户输入的主题}
【场景】{用户选择的场景}
【时长】{时长要求，默认3分钟约600字}
【风格】{风格偏好}
【补充要求】{补充说明}

要求：
1. 结构清晰：开场白（吸引注意）+ 正文（3个要点）+ 结尾（总结升华）
2. 语言口语化，适合朗读表达
3. 适当使用排比、设问等修辞手法
4. 标注重音词和停顿建议（用 **粗体** 标注重音，用 /// 标注停顿）
---

返回格式：JSON
{
  "title": "标题",
  "content": "完整文案（含标注）",
  "estimated_duration": "3min",
  "key_points": ["要点1", "要点2", "要点3"],
  "tips": "表达技巧提示"
}
```

#### 四种场景差异化Prompt设计

| 场景 | Prompt侧重 | 生成特征 |
|------|-----------|----------|
| 演讲文案 | 结构化、感染力、修辞手法 | 开场白+要点展开+结尾升华，标注重音停顿 |
| 短视频口播 | 短平快、钩子开头、节奏紧凑 | 前3秒抓注意、口语化、有互动感 |
| 直播带货话术 | 产品卖点、信任建立、促单转化 | 痛点+解决方案+FAB法则+逼单 |
| 万能开场白 | 场景覆盖广、模板化、即插即用 | 多种通用开场句式，用户填空即可 |

#### 免费次数策略

| 用户状态 | 每日免费次数 | 打卡奖励 | 上限 |
|----------|-------------|----------|------|
| 新用户（0-7天） | 3 次/天 | 打卡 +1 次 | 4 次/天 |
| 7天入门达成 | +1 次（永久） | 打卡 +1 次 | 5 次/天 |
| 30天进阶达成 | +3 次（永久） | 打卡 +1 次 | 7 次/天 |
| 60天达人达成 | +5 次（永久） | 打卡 +1 次 | 9 次/天 |
| 100天大师达成 | 不限次数 | — | ∞ |

> 次数每日 00:00 重置，不累加前一天余额

#### 交互规则

| 规则 | 说明 |
|------|------|
| 生成中状态 | 按钮loading动画 + 文案区域显示"正在为你创作中..." |
| 生成失败 | 不扣次数，提示重试。连续失败3次提示用户稍后再试 |
| 复制 | 一键复制去除标注符号的纯文本版本 |

| 历史记录 | 保存最近30条生成记录，更早的自动清理 |
| 收藏 | 收藏可永久保存，不受30条限制 |

## 7. 后台管理端功能需求

### 7.1 功能模块

```
后台管理系统（Web端，Vue3 + Flask）
├── 📊 数据看板
├── 📝 素材管理
├── 📋 打卡配置
├── 🤖 AI配置
├── 👥 用户管理
```

### 7.2 各模块详细需求

#### 数据看板
- **核心指标卡片**：日新增用户、日活(DAU)、打卡完成率、AI功能使用次数
- **趋势图表**：近30天用户增长曲线、每日打卡率变化、AI功能使用趋势
- **内容消费排行**：最热门训练题Top20、各分类练习次数分布
- **用户留存**：次日留存、7日留存、30日留存

#### 素材管理
- **训练题库管理**：CRUD训练题目，支持分类、难度、标签设置
- **范本音频上传**：上传/替换TTS合成的范本音频，支持批量导入
- **上下架控制**：单条素材可上架/下架，下架内容用户端不可见但数据保留
- **热门推荐配置**：手动配置首页"热门推荐"展示内容
- **新手专区配置**：编辑7天入门训练营的每日内容

#### 打卡配置
- **每日任务模板**：配置3个任务的类型、标题、难度、时长要求
- **任务排期**：支持按日期预设未来任务，支持循环模板
- **成长目标配置**：编辑7/30/60/100天目标的条件和奖励
- **特殊日期配置**：节假日可配置"免打卡"或"轻量任务"

#### AI配置
- **每日免费次数**：设置不同用户层级的AI生成次数
- **Prompt模板管理**：编辑4种场景的Prompt模板和参数
- **Qwen模型参数**：选择模型版本、调整temperature等参数
- **语音评测参数**：设置评分维度权重、基准分数线

#### 用户管理
- **用户列表**：查看所有用户，支持搜索、筛选
- **用户详情**：查看单用户打卡记录、练习记录、AI使用记录
- **权益管理**：手动调整用户AI次数、成长等级

#### 消息推送
- **模板消息**：支持微信订阅消息模板配置
- **定时推送**：每日练习提醒（默认20:00）、打卡成功通知
- **手动推送**：新素材上线通知、活动通知
- **推送记录**：查看历史推送记录和触达率
